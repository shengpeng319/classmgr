# ClassMgr UX 改进迭代 - Autoresearch 配置

## 实验目标
通过多轮迭代，持续改进 ClassMgr 的 UX 质量，直到达到生产级标准。

## 评审标准（每轮必须满足）

### 必须满足
1. **功能完整性** - 所有 TabBar 页面必须有实质功能
2. **数据一致性** - 前后端数据模型匹配
3. **移动端可用性** - 触摸区域 ≥ 44px，按钮间距合理
4. **视觉层次** - 重要操作有视觉引导，状态清晰

### 建议满足
1. **动画流畅** - 过渡动画 200-300ms
2. **空状态设计** - 有意义的空状态提示
3. **加载状态** - 请求时有 loading 反馈
4. **错误处理** - 网络错误有 toast 提示

## 迭代流程

### Round 1 (当前)
- **Coder (Emily)**: 
  - 完成 lottery.vue 核心功能（抽卡逻辑、卡池展示）
  - 检查其他页面 UI 问题
- **Main (UX Review)**:
  - 使用 ui-ux-pro-max skill 审核所有页面
  - 列出必须修改项
- **Researcher (我)**:
  - 使用 autoresearch 审批每轮结果

### 验收条件
- lottery.vue 有完整抽卡流程
- 所有页面无明显 UX 问题
- 代码符合 Uni-app 规范

## 评分体系
- 10分：达到生产级标准
- 8-9分：良好，有小问题可接受
- 6-7分：基本功能可用，需改进
- <6分：不合格，重做
